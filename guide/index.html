프롬프트 작성법

프롬프트를 몰라도 쓸 수 있게, 최소한의 원칙만 정리했습니다.

원칙 1

상황을 먼저 써라

프롬프트를 잘 쓰는 사람은 어려운 말을 쓰는 사람이 아니라, 지금 뭘 해결하려는지 분명하게 적는 사람입니다.

원칙 2

원하는 출력 형식을 적어라

bullet, 표, 메일, 회의록처럼 결과 모양을 함께 적으면 훨씬 안정적으로 나옵니다.

원칙 3

변수만 바꿔 돌려라

잘 되는 프롬프트는 다시 쓰기 쉬워야 합니다. 고정 문장과 바뀌는 변수만 분리하세요.

좋은 프롬프트 기본 구조

1. 역할
어떤 역할로 답할지 지정
2. 상황
무슨 문제를 해결하는지 설명
3. 원하는 결과
메일, 회의록, 요약, 문구 등 출력 형식 지정
4. 변수
상황마다 바뀌는 값만 따로 넣기

바로 써볼 예시

  • 회의록 프롬프트 예시 7개 — 회의가 끝난 뒤 발언 요약, 액션 아이템, 일정까지 한 번에 정리하고 싶을 때 쓰는 프롬프트 예시 모음입니다.
  • 메일 답장 프롬프트 모음 — 답장을 빨리 보내야 하는데 예의는 지키고 싶을 때 쓰는 업무 메일 프롬프트입니다.
  • 고객문의 답변 프롬프트 — 고객 문의에 답변할 때 말투를 부드럽게 유지하면서도 필요한 정보를 정확히 전달하고 싶을 때 쓰는 프롬프트입니다.
